Fuel Model Export

Once a custom fuel model has been saved in BehavePlus format (File > Save as a fuel model > BehavePlus format), it can then be saved in a format that can be read by NEXUS, FlamMap, and FARSITE using File > Save as a fuel model > FARSITE format, English or File > Save as a fuel model > FARSITE format, metric. Select (click on or highlight) one or more of the fuel models in the list and press Ok to write the selected fuel models to a file.

NOTE: Before you can export a BehavePlus custom fuel model, it must be saved and attached to the current BehavePlus session.

BehavePlus format vs FARSITE format: Custom fuel models you create and save in the BehavePlus ".bpf" format can be exported to the".fmd" format used by NEXUS, FlamMap, and FARSITE. Files with a ".fmd" extension cannot be used in BehavePlus, and ".bpf" files cannot be used in NEXUS, FlamMap, or FARSITE.

Naming convention: In the exported ".fmd" file the first field for each fuel model is the fuel model number. This number is defined in the custom fuel model worksheet. These numbers should match the data in the landscape file used in FlamMap or FARSITE. See Fuel Model Number or page 5 of Scott and Burgan's (2005) fuel model guide for available numbers. The second field is the fuel model code, which can be up to seven characters long. It is also defined in the custom fuel model worksheet. The fuel model description that appears when you attach the fuel model to BehavePlus is taken from the Description field at the top of the BehavePlus worksheet.

Files exported in ".fmd" format will be saved using the following units.
